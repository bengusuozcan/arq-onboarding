Navigation & Sidebar

Finding your way around the Notion workspace

Sidebar Essentials

  • Teamspaces — shared areas for different groups (e.g., "Arq" teamspace)
  • Favorites — pin pages you use daily (star icon)
  • Private — your personal pages, only visible to you
  • Search — Cmd+K to instantly find any page

Quick Tips

  • Hover over sidebar items to see + and ... menus
  • Drag pages to reorder or nest them
  • Toggle sidebar with Cmd+\

Blocks & Pages

The building blocks of everything in Notion

Slash Commands

Type / anywhere to see all block types:

  • /heading — H1, H2, H3
  • /bullet — Bullet list
  • /todo — Checkbox
  • /toggle — Collapsible section
  • /callout — Highlighted box
  • /table — Simple table
  • /database — Full database
🔄
Transform any block — click the ⋮⋮ handle → "Turn into" to change block types.

Creating Pages on the Fly

Create and link pages without leaving your current page

The @ Method

  1. Type @ followed by a page name
  2. If it doesn't exist, select "Create new page"
  3. A linked page is created instantly — click to open
  4. Great for referencing contacts in meeting notes

Other Quick Methods

  • Cmd+N — new page in sidebar
  • Type /page — inline sub-page
  • + button in sidebar — new page in teamspace

Databases — The Core Concept

Databases are the backbone of how Arq organizes information in Notion

What Is a Database?

A collection of pages with structured properties. Each row is a full page.

Key Features

  • Properties — typed fields: text, select, date, person, relation, formula
  • Views — same data, different layouts:
    • Table Spreadsheet-like
    • Board Kanban columns
    • Calendar Date-based
    • List Compact list
  • Filters & Sorts — show only what matters
  • Groups — cluster by category, status, owner

Watch: Databases in Notion

📈
Relations are the superpower — they let you link entries across databases. A Contact links to their Interactions, an Interaction links to its Project.

Arq Databases: How They Connect

Our databases are linked together to build institutional memory

Contacts (CRM) ←— External Participants —→ Interactions ←— Related Project —→ Projects

Real Examples

  • You attend a meeting:
    Create an Interaction → link the Contact as External Participant → set the Related Project
  • You check a contact's history:
    Open a Contact → see every Interaction they've been part of (the relation shows on both sides)
  • You review a project:
    Open a Project → see all linked Interactions and which Contacts were involved

Our Key Databases

  • Contacts — our CRM. People we engage with, tagged and linked to organizations.
  • Organizations — institutions, companies, think tanks.
  • Interactions — every meeting, call, email, event. Links to External Participants and Related Project.
  • Projects — active workstreams that tie Interactions together.

Claude <> Notion

Connect your LLM to Notion — this is how we supercharge our workflow

Why This Matters

At Arq, we use Claude as our AI layer. By connecting Claude to Notion via MCP (Model Context Protocol), Claude can:

  • Search your Notion workspace — find pages, contacts, meetings
  • Read any page or database — get full content and properties
  • Create new pages, tasks, and database entries
  • Update existing pages — change properties, add content
  • Query databases — filter and sort without opening Notion
⚡
Example: "Add a new contact: Jane Smith, European Commission, DG CNECT" — Claude creates the entry in Notion with the right properties, tags, and organization link.

How to Set Up

Setting up the Claude <> Notion MCP connection is straightforward. Just search online for "Claude MCP Notion setup" or ask Claude to guide you — it'll walk you through it step by step.

What Notion AI Can Do

Notion has built-in AI that works directly inside your pages and databases:

  • Summarize meeting notes into key takeaways
  • Draft content — emails, briefs, descriptions
  • Extract action items automatically
  • Fill databases — auto-populate properties from page content
  • Ask questions about your workspace

What Lives Where

Our knowledge lives across four systems — here's the map. This structure is currently still being revised as we grow.

SystemWhat Goes HereWho Uses It
Notion Contacts (CRM), meetings, tasks, project boards, collaboration Everyone
Obsidian (Intel Vault) Research profiles, deep analysis, event intel, publications Research & strategy team
GitHub Operations docs, processes, tool guides, templates, code Everyone (via Claude Code)
Google Docs Real-time collaborative drafts, external-facing documents Everyone
💡
Rule of thumb: Structured data & daily work → Notion  |  Research intel → Obsidian  |  Processes & code → GitHub  |  Collaborative writing → Google Docs
🤖
Claude Code is the AI layer that connects all four systems. You can ask it to search across Notion, Obsidian, and GitHub, create entries, update records, and answer questions from any source.
