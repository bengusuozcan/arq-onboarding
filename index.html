Coda → Notion Mental Model

Key concepts mapped from what you already know

Coda ConceptNotion EquivalentKey Difference
DocPagePages nest infinitely — every page can contain sub-pages
TableDatabaseDatabases have multiple views (table, board, calendar, gallery, list)
Section / FolderTeamspace / Page treeTeamspaces control access; page tree controls organization
Row detailPage inside databaseEvery database row IS a full page — rich content inside each row
ViewDatabase ViewSame data, different layouts — filtered, sorted, grouped
Button / AutomationButton block / AutomationSimilar concept, different UI
Formula columnFormula propertyDifferent syntax but same idea
💡
Biggest mindset shift: In Notion, everything is a block. Text, images, tables, embeds, databases — all blocks. You can drag, rearrange, and transform any block into another type.

Navigation & Sidebar

Finding your way around the Notion workspace

Sidebar Essentials

  • Teamspaces — shared areas for different groups (e.g., "Arq" teamspace)
  • Favorites — pin pages you use daily (star icon)
  • Private — your personal pages, only visible to you
  • Search — Cmd+K to instantly find any page

Quick Tips

  • Hover over sidebar items to see + and ... menus
  • Drag pages to reorder or nest them
  • Toggle sidebar with Cmd+\

Creating Pages on the Fly

Speed up your workflow — create pages without leaving your current page

The @ Method

  1. Type @ followed by a page name
  2. If it doesn't exist, select "Create new page"
  3. A linked page is created instantly — click to open
  4. Use this inside meeting notes to quickly reference or create people/topic pages

Other Quick Methods

  • Cmd+N — new page in sidebar
  • Type /page — inline sub-page
  • + button in sidebar — new page in teamspace
  • Drag text → turn into a page for quick refactoring
🔗
Linking tip: Use @ to link to existing pages too — great for cross-referencing contacts in meeting notes.

Databases — The Core Concept

Databases are the backbone of how Arq organizes information in Notion

What Is a Database?

A collection of pages with structured properties. Each row is a full page.

Key Features

  • Properties — typed fields: text, select, multi-select, date, person, relation, formula
  • Views — same data, different layouts:
    • Table Spreadsheet-like
    • Board Kanban columns
    • Calendar Date-based
    • List Compact list
  • Filters — show only what matters (e.g., "my tasks", "this week's meetings")
  • Sorts — order by any property
  • Groups — cluster by category, status, owner, etc.

Coming from Coda tables?

Arq Database: Contacts

Our lightweight CRM — track people we engage with

What It Tracks

  • Name — full name of the contact
  • Organization — linked to our Organizations DB
  • Role / Title — their position
  • Tags — multi-select for categorization (e.g., EU Parliament, AI Safety, Think Tank)
  • Interactions — relation to meetings/events where we met them
  • Owner — Arq team member who manages the relationship
  • Notes — quick context visible in table view

How to Add a Contact

  1. Open the Contacts database (link below)
  2. Click "+ New" to add a row
  3. Fill in name, org, role, and tags
  4. Open the row page to add detailed notes

Arq Database: Interactions

Unified database for all meetings — internal team calls & external stakeholder meetings

What It Tracks

  • Title — descriptive meeting name
  • Date — when it happened
  • Type — Internal / External / Event
  • Participants — linked contacts + team members
  • Related Organizations — linked orgs
  • Notes — meeting content, action items, key takeaways
  • Follow-up Tasks — linked to Spinoff Tasks DB

How to Log a Meeting

  1. Open the Interactions database
  2. Click "+ New"
  3. Set the title, date, and type
  4. Link participants (from Contacts DB)
  5. Open the page and write your notes
  6. Create follow-up tasks if needed

Arq Database: Spinoff Tasks

Track action items, follow-ups, and project tasks

Key Properties

  • Task Name — clear, actionable description
  • Owner — who's responsible
  • Priority Level — High / Medium / Low
  • Phase — workflow stage (e.g., "Phase 1: Post-Disclosure")
  • Status — Not started / In progress / Done
  • Due Date — deadline
  • Source Interaction — linked meeting that generated this task

How to Create a Task

  1. Open the Spinoff Tasks board
  2. Click "+ New" in the relevant status column
  3. Set name, owner, priority, and due date
  4. Link to source interaction if applicable

Daily Workflows

Three real scenarios you'll encounter

📅 After a Meeting

  1. Open Interactions DB → + New
  2. Set title, date, type, participants
  3. Write notes with key takeaways & action items
  4. For new people: add to Contacts DB with tags
  5. Create follow-up tasks in Spinoff Tasks

🔍 Looking Up a Person

  1. Press Cmd+K and search their name
  2. Check their Contact page for role, org, tags
  3. Click linked Interactions to see past meetings
  4. Check the Intel vault (Obsidian) for deeper research notes

✅ Managing Your Tasks

  1. Open Spinoff Tasks
  2. Filter by Owner = You
  3. Sort by due date or priority
  4. Drag tasks across columns as you complete them
  5. Open a task page for detailed notes
⏰
Time saver: Save your filtered views! Click "Save view" after filtering — next time you open the database, your custom view is one click away.

What Lives Where

Our knowledge lives across four systems — here's the map

SystemWhat Goes HereWho Uses ItAccess
Notion Contacts (CRM), meetings, tasks, project boards, team collaboration Everyone Browser / desktop app
Obsidian (Intel Vault) Research profiles (people, orgs, topics), deep analysis, event intel, publications Research & strategy team Obsidian app + Claude Code
GitHub (this repo) Operations docs, processes, tool guides, templates, code projects Everyone (via Claude Code) Git / Claude Code
Google Docs Real-time collaborative drafts, external-facing documents, shared editing Everyone Browser
💡
Rule of thumb:
  • Structured data & daily work → Notion
  • Research intel & strategic analysis → Obsidian
  • Processes, templates, code → GitHub
  • Collaborative writing → Google Docs
🤖
Claude Code is the AI layer that connects all four systems. You can ask it to search across Notion, Obsidian, and GitHub, create entries, update records, and answer questions using information from any source.
